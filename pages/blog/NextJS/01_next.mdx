`Next.js`는 공식 doc에서 위와 같이 소개하고있다. 의미를 해석해보자.

- `Full-stack` : 프론트 / 백엔드단의 기능을 모두 가지고 있음
- `Web-Application` : 웹에서 동작하는 웹페이지 / 어플리케이션
- `Framework` : 주된 기능을 직접 구현하지않고, 정해진 규격대로 코드를 작성

## 렌더링 방식

### 클라이언트 사이드 렌더링 (CSR)

![sample](/img/2024-05-30-20-55-37.png)

리액트는 기본적으로 클라이언트 사이드 렌더링으로 동작한다.

1. 서버가 브라우저를 응답을 보냄
2. 브라우저가 `JavaScript`를 다운받음
3. 브라우저가 `React`를 실행
4. 사용자가 사용 가능

클라이언트 사이드는 위와 같은 흐름으로 동작하는데, 이걸 통해 우리가 알 수 있는 점이 있다.

- `CSR`은 `JavaScript`가 없을경우 렌더링 되지 않는다.

이는 다르게 말하자면 검색엔진 크롤링의 `robot.txt` 가 `html`을 인식할 수 없다는 문제점이다. 따라서 검색엔진에서 거의 노출되지 않는다.

- `CSR`는 초기 로딩속도가 느리지만, 이후 아주 빠르게 동작한다.

`SSR` 대비 더 빠른 속도로 작동할 수 있다. 다만 컴포넌트가 많고 페이지가 복잡해질수록 초기 로딩시간이 길어질 수 있다.

### 서버 사이드 렌더링 (SSR)

![sample](/img/2024-05-30-20-55-53.png)
> Image : [The Benefits of Server Side Rendering Over Client Side Rendering](https://medium.com/walmartglobaltech/the-benefits-of-server-side-rendering-over-client-side-rendering-5d07ff2cefe8) 

`Next.js`는 서버사이드 렌더링으로 동작한다.

1. 서버가 렌더링된 HTML의 응답을 브라우저에 전송
2. 자바스크립트 다운로드 진행
3. 브라우저가 React를 진행
4. 사용자가 사용가능

시작지점에 HTML응답을 처리해서 전송해줘 사용자가 화면을 볼수는 있다. 여기서 알 수 있는 점은

- `SSR`은 `JavaScript`가 없을경우에도 화면을 렌더링해준다.

검색엔진 크롤링이 정상적으로 동작하고, 자바스크립트가 느린 환경의 사용자도 화면의 렌더링은 미리 진행되어서 사용자친화적으로 동작한다는 의미다.

- 사용자가 요청을 보낸 페이지에 대해 요청이 들어올때마다 처리한다.

미리 로딩을 하는게 아니라, 사용자가 요청을 보낸 페이지에따라 요청을 처리해서 HTML을 다운로드 한다. 따라서 코드가 작게 분할되어 빠른 작동속도를 보여줄 수 있다. 이는 다르게 말하면 약간의 인풋 렉은 존재할 수 있다.

## 요약

| 분류 | 장점 | 단점 |
| --- | --- | --- |
| `CSR` | - 초기 로딩후 빠른 상호작용 <br/> - 서버 부하 감소 | - 초기 로딩 지연 <br/> - `SEO`(검색엔진 최적화) 불가능 |
| `SSR` | - `SEO`(검색엔진 최적화)로 노출 증가 <br/> - 초기 로딩속도가 빠르고 빠른 상호작용 | - 서버 부하 증가 <br/> - 환경에 따라 인풋 렉 존재 |
